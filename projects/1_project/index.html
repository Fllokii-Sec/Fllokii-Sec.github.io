SOC Homelab

Wazuh Siem integrated with pfsense, sysmon, suricata, and VirusTotal, with simulation attack.

Soc Home Lab

Objective

This project demonstrates the design, deployment, and validation of a localized Security Operations Center (SOC) home lab ecosystem. The primary objective was to show you can build a comprehensive telemetry catchment architecture to aggregate endpoint logs, network events, and signature-based Intrusion Detection System (IDS) alerts into a centralized SIEM platform for analysis and security orchestration with open-source tools, with the enhancement of Virus Total and file integrity monitoring for even more detection capabilities.

Skills Learned

  • Advanced understanding of SIEM concepts and practical application.
  • Proficiency in analyzing and interpreting network logs.
  • Ability to generate and recognize attack signatures and patterns.
  • Enhanced knowledge of network protocols and security vulnerabilities.
  • Development of critical thinking and problem-solving skills in cybersecurity.

Tools Used

  • Security Information and Event Management (SIEM) system for log ingestion and analysis.
  • PFsense for Edge firewalling, network isolation, and perimeter access logging.
  • Suricata for Signature-based network threat detection and traffic profiling.
  • Sysmon for Advanced system logging (Process creation, network connections, registry changes).
  • VirusTotal for Automated file hash reputation lookup and analysis orchestration.

Steps

Step 1: SIEM Management Setup (Ubuntu Server) Objective: Establish a highly stable, hardened log aggregation and analysis engine.

Implementation: Created an Ubuntu Linux virtual machine allocated with a static IP configuration to prevent communication drops from endpoints. Executed the automated installation script (curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh && sudo bash ./wazuh-install.sh -a) for the unified Wazuh Manager architecture.

The “Why”: Centralization is the foundation of any SOC. By decoupling the manager from the endpoints, we ensure that if a target host is completely compromised or destroyed by ransomware, its forensic trail remains securely stored off-host in the SIEM engine.

Step 2: Sysmon Set Up And Integration Objective: Elevate default Windows logging to capture granular kernel-level event data.

Implementation:

  • Installed the modular Wazuh Windows Agent executable and registered it using the secure pre-shared key generated by the Manager node.
  • Deployed Microsoft Sysmon utilizing a customized, highly regarded configuration (e.g., SwiftOnSecurity schema) to filter out standard OS noise while capturing high-value Event IDs.

The “Why”: Default Windows Event Viewer lacks the context required for deep incident response. Adding Sysmon exposes process lineage, command-line arguments, and process hollowing indicators (such as Event ID 1: Process Creation, or Event ID 3: Network Connection) directly into our SIEM engine.

Step3: Perimeter Security & Firewall Log Integration (pfSense) Objective: Gain complete visibility over ingress and egress traffic moving through the network boundaries.

Implementation: Configured a virtualized pfSense instance acting as the primary gateway for the lab environment. Established custom firewall rules to block unauthorized lateral movement and configured the system’s remote logging features to ship system messages via syslog forwarders straight to the Wazuh Manager port.

Step 4: Host-Based Intrusion Detection System Deployed (Suricata) Objective: Implement signature-based deep packet inspection directly at the host level.

Implementation: Installed Suricata as a local host-based agent on the Windows target. Wired the local Suricata engine’s eve.json output into the Wazuh agent’s custom log collection directives (ossec.conf) so that any network alerts are ingested into the manager in real time.

Step 5: Threat Intelligence & API Integration (VirusTotal and File Integrity Monitoring) Objective: Automate threat validation workflows and implement real-time File Integrity Monitoring (FIM) to drastically lower the Mean Time to Detect (MTTD)..

Key Takeaways & Security Mindset Acquired

  • Visibility is Nothing Without Context: Collecting logs from pfSense or Windows independently provides a fractured narrative. True detection capability was realized when correlating the network event (Suricata/pfSense alerting on high-volume SSH connections) with the host event (Windows reporting rapid credential failures).

  • The Attacker’s Advantage vs. Defender’s Strategy: Simulating the attack via Kali reinforced how noisy automated brute-force tools are. As a defender, mapping these actions directly to the MITRE ATT&CK Framework (T1110 - Brute Force) allows for targeted, proactive defense strategies rather than reactive firefighting.

  • Open Source Security: How Open Source Tools not only can be a viable option for security and monitoring, but also a good source to get hands on practical skills in an isolated home enviroment to continue learning and becoming more knowlegable how ceartain fundementals tie together.
